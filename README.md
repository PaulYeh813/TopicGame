### 功能性需求:
###### 1.角色和敵人能夠順暢移動
###### 2.Hitbox判定：受到傷害或者觸發物件條件設定
###### 3.轉場：達到畫面邊界時背景及平台能變化
###### 4.血量：判斷主角或敵人是否存活
###### 5.觸發事件：例如:碰到特定物件、持有某項道具才能通過
###### 6.存檔功能
###### 7.使用者系統設定
### 非功能性需求:
###### 1.主角、敵人動畫
###### 2.平台材質
###### 3.前景、背景設計
###### 4.UI設計
###### 5.降低硬體需求
